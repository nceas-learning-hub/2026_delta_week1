Style guide preferences:

* Snake case for file names and variable names
* Title case for headings and titles
    * This includes titles for callouts?
* Learning objectives:
    * For content driven lessons (most things) use verbs from Bloom's Taxonomy
        * https://en.wikipedia.org/wiki/Bloom%27s_taxonomy#/media/File:Bloom%E2%80%99s_Taxonomy_Verbs.png
    * For setup lessons (just accomplish a task) OK to use more direct actions as verbs (e.g., set, configure, define)
* Code chunks
    * Functional code chunks for students to view: echo: true
    * Functional code chunks behind the scenes: echo: false
    * Code chunks for students to view but that don't need to run:
        * ```r
        * ```{r} with #| eval: false
        * ```{{r}} to show code chunk header and footer
    * comments at end of line vs. footnotes?
* Naming code chunks: good thing!
* Tabsets: great for alternative IDE or methods
    * Avoid putting headers inside the tabsets (aside from labeling the tabsets appropriately) - assuming section numbering is enabled, the numbering gets weird
* Images: use ![](path) mode instead of <img src="path"> mode?  strengths? weaknesses?
    * ![]{fig-align="center" width="70%" .lightbox fig-alt="alternative text for accessibility"}
    * use those options and others as you see fit!
* External links: use {.external target="_blank" rel="noopener"}
    * .external makes a little "leaving this site" icon by the link
    * target="_blank" opens the link in a new tab, leaving the course page still easily available
    * rel="noopener" (sometimes automatic with target="_blank") is for security reasons
